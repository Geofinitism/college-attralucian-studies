## Core trajectory

Cognitive overload is reframed as a mathematical inevitability: context generation grows combinatorially (b^n) while memory decays exponentially (e^{-t/τ}), producing a persistently positive derivative of active cognitive load that cascades into thrashing — and the same structural problem appears in LLM context windows, making deliberate management protocols a shared necessity for both human and machine cognition.

## Key claim

Scaling overload — the state where d|C(t)|/dt > 0 persistently — is not a psychological failure but a structural consequence of fractal input growth outpacing exponential decay; it is addressable only through deliberate management: triage, pruning, and spaced repetition scheduling.

## Key evidence

- Context generation: |C_potential(t)| ∝ b^n — each active context spawns sub-contexts at branching factor b > 1
- Memory decay: P_retention(t) = e^{-t/τ_i} — the forgetting curve, with nested decay constants for short-term, working, and long-term memory
- Scaling overload equation: d|C(t)|/dt = R_gen(t) − R_decay(t); overload when derivative is persistently positive
- Pruning heuristic: IF t_last_active > 3τ_i AND w_i < w_threshold, THEN PRUNE c_i
- Spaced repetition: I_{n+1} = I_n · f — exponential growth of review intervals counteracts exponential decay
- LLM analogy: context window as bounded memory; token decay as sliding window forgetting; both systems exhibit thrashing under overload
